# 项目各层说明
父项目backed包含以下4部分：
1. component--框架组件包
   1. common-core--底层核心类库及框架依赖
   2. common-dao --底层数据层类库及框架依赖(dao独立并且统一封装分页查询的请求和结果类,便于解耦使用不同的数据层技术;dao与core分离便于一些不依赖db的项目)
2. platform--基础服务,支持多租户
   1. authentication--认证服务
   2. authority--权限服务
   3. 其他服务，比如文件服务，数据字典服务等，或者是其他业务中台下沉的服务，比如消息中心，支付中心等
3. services--业务系统，依赖component，可能会使用到基础服务
   1. book--简单的图书管理系统demo
   2. 至于book系统中的dao层、service层也可以单独使用微服务，一般建议放在一个系统中，保证领域代码集中。
4. api--是服务之间接口调用层，单独抽离出来，解耦方便各业务线项目使用

#  book系统功能简介
1. 图书的增、删、改、查功能，支持列表分页和用户角色权限控制
2. 框架支持登录权限控制、分层、日志、缓存、数据库(H2与mysql自由切换)、统一异常管理、统一返回体控制、基于环境的配置分离、swagger等。
